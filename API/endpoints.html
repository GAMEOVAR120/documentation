Endpoints

V13 Preview

This is a preview of the available API routes in MCSS v13.

Work In Progress

This section of the documentation is a work in progress. It is not complete and may change at any time.
The API is still in development. It is NOT perfect and will be improved in the future.

This guide is aimed at developers

The following tips and information is aimed at developers wanting to create apps that utilize the API. If you are a regular user this may not be for you. As such some steps will assume that you already have basic knowledge of how API calls work and that you can read a documentation.

Or if you are looking for the API routes for V12.

General​

Get API Version​

Route
GET /api/v1

Description
Return the api version.

Authentication
API Key

Server​

Get Server​

Route
GET /api/v1/servers/:serverId

Description
Return info of a specific server.

Authentication
API Key

Path Parameters

// ServerID (required)
:serverId

// example 
bc730283-4033-43e3-a8f8-f511606bce2a

URL Parameters

// Filter Parameter (optional)
// Additional filter to specify how much data you want to fetch.
filter: None, Minimal, Status
// None: Default, show all server info (serverId, status, name, description, pathToFolder, folderName, creationDate, isSetToAutoStart, keepOnline, javaAllocatedMemory, javaStartupLine)
// Minimal: Returns serverId, status, name, description & creationDate
// Status: Returns serverId & status

// example 
?filter=Minimal

Run Server Action​

Route
POST /api/v1/servers/:serverId/execute/action

Description
Change the status of a specific server.

Authentication
API Key

Path Parameters

// ServerID (required)
:serverId: guid

// example 
bc730283-4033-43e3-a8f8-f511606bce2a

Request Body

// Action (required)
action: integer
// 0 -> does nothing
// 1 or Stop
// 2 or Start
// 3 or Kill
// 4 or Restart

// example 
{
  "action" : 1
}

Console​

Get Server Console​

Route
GET /api/v1/servers/:serverId/console

Description
Return the server console.

Authentication
API Key

Path Parameters

// ServerID (required)
:serverId

// example 
bc730283-4033-43e3-a8f8-f511606bce2a

URL Parameters

// AmountOfLines Parameter (required)
// Amount of lines you want to fetch, -1 to fetch everything.
amountOfLines: integer

// TakeFromBeginning Parameter (optional)
// Specify if you want to start fetching from the oldest lines first.
takeFromBeginning: boolean

// Reversed Parameter (optional)
// Option to flip the order of the returned console lines.
reversed: boolean

// example 
?amountOfLines=50?takeFromBeginning=false?reversed=false

Check if Console is outdated​

Route
GET /api/v1/servers/:serverId/console/outdated

Description
Check if your version of the console is outdated. This by sending the last 2 last lines for comparison.

Authentication
API Key

Path Parameters

// ServerID (required)
:serverId

// example 
bc730283-4033-43e3-a8f8-f511606bce2a

URL Parameters

// SecondLastLine Parameter (required)
secondLastLine: string

// LastLine Parameter (required)
lastLine: string


// example 
// secondLastLine: [19:25:31] [Server thread/INFO]: Closing Thread Pool
// lastLine: [19:25:31] [Server thread/INFO]: Closing Server
// after URI Encoding the Parameters, it will look like:
?secondLastLine=%5B19%3A25%3A31%5D%20%5BServer%20thread%2FINFO%5D%3A%20Closing%20Thread%20Pool?lastLine=%5B19%3A25%3A31%5D%20%5BServer%20thread%2FINFO%5D%3A%20Closing%20Server

Create Scheduler Task​

Route
POST /api/v1/servers/:serverId/scheduler/tasks

Description
Create a new task.

Authentication
API Key

Path Parameters

// ServerID (required)
:serverId

// example 
bc730283-4033-43e3-a8f8-f511606bce2a

Request Body

// Name (required)
name: string

// Enabled (optional)
enabled: boolean

// Timing (required)
timing: object

// Job (required)
job: object

Timing

// FixedTime
repeat: boolean
timeSpan: TimeSpan

// Interval
repeat: boolean
interval: integer

// Timeless
// no parameters

Job

// Server Action
action : ServerAction
// 0 or Stop
// 1 or Start
// 2 or Kill
// 3 or Restart

// Run Commands
commands: string[]

// Start Backup
backupIdentifier: guid
// example (Interval & Run Commands)
{
  "name": "Made with the API <3",
  "enabled": true,
  "timing": {
      "repeat": true,
      "interval": 150
  },
  "job": {
      "commands": [
        "/say hello from the api"
    ]
  }
}

Edit Scheduler Task​

Route
PUT /api/v1/servers/:serverId/scheduler/tasks/:taskId

Description
Update a specific server task.
You can not change the timing or job of the original task, only its values.

Authentication
API Key

Path Parameters

// ServerID (required)
:serverId
// example 
bc730283-4033-43e3-a8f8-f511606bce2a

// TaskId (required)
:taskId
// example 
8ce91a2f-5b99-486b-bf3d-af823a999824

Request Body

// Name (optional)
name: string

// Enabled (optional)
enabled: boolean

// Timing (optional)
timing: object

// Job (optional)
job: object
// example (Interval & Run Commands)
{
  "name": "Made with the API <3 V2",
  "enabled": true,
  "timing": {
      "repeat": true,
      "interval": 200
  },
  "job": {
      "commands": [
        "/say hello from the api",
        "/say this a second command"
    ]
  }
}
